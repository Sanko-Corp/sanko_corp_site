# Handoff: 三交設備株式会社 コーポレートサイト

## Overview
配管設備工事会社「三交設備株式会社」（関西圏拠点）の5ページ構成コーポレートサイト。
給排水・消火・空調の配管工事の事業紹介、品質管理方針、採用情報、お問い合わせを掲載し、
BtoBの取引先・求職者・地域住民の3層をターゲットにした信頼感ある企業サイト。

ページ構成:
1. **Home** — ヒーロー、会社の理念、3つの主要ページへの遷移カード、実績バンド
2. **事業内容（Business）** — 3つの工事領域の紹介
3. **品質管理（Quality）** — 品質管理方針2項目
4. **採用情報（Recruit）** — 求める人物像、働く3つの魅力、募集要項
5. **お問い合わせ（Contact）** — 連絡先情報＋問い合わせフォーム＋地図

## About the Design Files
このバンドル内のファイルは **HTMLで作成されたデザインリファレンス** です。意図した
見た目と挙動を示すプロトタイプであり、そのまま本番コードとしてコピーするものではありません。
タスクは、これらのHTMLデザインを **ターゲットとなるコードベースの既存環境
（React / Vue / Next.js / 静的サイトジェネレータ等）で、その確立されたパターンと
ライブラリを使って再現すること** です。まだ環境が存在しない場合は、本プロジェクトに
最適なフレームワークを選定して実装してください。

> 注: `.dc.html` ファイルは社内デザインツールのコンポーネント形式（`<x-dc>` テンプレート＋
> `class Component extends DCLogic` のロジック）で書かれています。`support.js` はそのランタイムで、
> **本番には不要** です。マークアップ・スタイル・挙動の「仕様書」として読んでください。

## Interactions & Behavior
- **ページ遷移**: SPA的に状態 `page` を切り替え（'home' | 'business' | 'quality' | 'recruit' | 'contact'）。遷移時に `window.scrollTo(0,0)`。各ページコンテンツは `animation: fadeUp .5s ease both`（18px下から、opacity 0→1）でフェードイン。
  - 本番では通常のルーティング（React Router / Next.js のページ等、URLを持つ）に置き換えることを推奨。各ページに固有URL（`/`, `/business`, `/quality`, `/recruit`, `/contact`）を与える。
- **ナビ アクティブ表示**: 現在ページのボタンに文字色 #15639e ＋ 下線。
- **カード hover**: shadow強化 ＋ translateY(-4px)。リンク「詳しく見る →」は hoverで gap が 6px→11px に広がる。
- **ボタン hover**: 各bgが濃いトーンへ。
- **フォーム挙動**:
  - フィールド: お名前（必須）/ メールアドレス（必須）/ 種別（select: 工事のご相談・お見積り / 保守・点検・修理について / 採用への応募 / その他）/ お問い合わせ内容（textarea, 必須）
  - 送信時バリデーション: 名前・メール・内容のいずれかが空（trim後）なら、エラーメッセージ「必須項目（お名前・メール・内容）をご入力ください。」を #d4543a で表示。
  - バリデーション通過で `sent=true` に切替→ フォームを成功メッセージ（緑カード「✓ 送信しました」）に置換、`scrollTo(0,0)`。
  - 入力欄 focus時は border-color #2e8fcf。
  - **重要**: 現状のプロトタイプは実送信しないモック。本番では実際のメール送信/フォームバックエンド（API、メール送信サービス、reCAPTCHA等）の実装が必要。

## State Management
- `page` — 現在表示中のページ（文字列）。本番ではURLルーティングで代替推奨。
- `fName`, `fEmail`, `fMsg`, `fType` — フォーム入力値。
- `sent` — 送信完了フラグ（true でサンクスメッセージ表示）。
- `showError` — バリデーションエラー表示フラグ。
- データ取得要件: 現状なし（静的）。将来、採用要項やお知らせをCMS化する場合はその限り。
